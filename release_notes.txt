## JellyRip v1.0.22 Release Notes (UNSTABLE)

JellyRip v1.0.22 — deep audit cleanup.  Many small correctness +
ergonomics improvements across the engine, settings UI, and test
suite.  No functional regressions; the bundled ripper, validator,
and library organizer behave identically to v1.0.21 under default
settings.

### Highlights

Engine
- stabilize_timeout is now an actual deadline (was a single 1s
  sample) — polls every 0.1–1.0s up to the configured timeout,
  honors abort_event between samples.
- Drive-probe retries default bumped from 3 to 5.
- Dead scan_disc delegate removed (was shadowed by the live
  inline implementation).
- 5 magic numbers named as module-level constants.
- print() in engine layer routed through stdlib logging.

UI / Settings
- Pages docs site nav now works (baseurl set in _config.yml).
- Settings tab persist failures now log instead of swallowing.
- Utility chip failures surface in status bar + session log.

Workflow
- Partial-extras-move failures now flag the session as partial.
- Dead resume scaffolding removed from _run_disc_inner.

Tests
- 3 of 5 truncated test bodies reconstructed.
- 2 of 5 skipped security tests reframed as positive Qt-side
  tests.
- Test suite: 1645 passed, 8 skipped (was 1642 / 11).

Documentation hygiene
- Mojibake cleared from CHANGELOG.md, smoke-report-2026-05-04.md,
  and tests/test_behavior_guards.py.

### What's NOT in this release

No new ripping/validation/organization features.  Engine behavior
under default settings is byte-identical to v1.0.21.
