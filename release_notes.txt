## JellyRip v1.0.23 Release Notes (UNSTABLE)

JellyRip v1.0.23 — bug-fix release on top of the v1.0.22 audit
cleanup.  Two user-facing fixes (disc-scan responsiveness + tool
auto-detection); no new features.

- Direct download: https://github.com/unexpear/JellyRip/releases/download/v1.0.23/JellyRip.exe
- Installer: https://github.com/unexpear/JellyRip/releases/download/v1.0.23/JellyRipInstaller.exe

- Project site: https://unexpear.github.io/JellyRip/

### Highlights

Engine
- Stop is now responsive during disc scan.  MakeMKV scan output
  feeds through a reader thread + queue (proc.poll()) instead of a
  blocking readline(), so the Stop button takes effect promptly and
  trailing title metadata isn't dropped.
- Blank tool path no longer breaks FFmpeg/ffprobe auto-detect.  An
  empty ffprobe_path/makemkvcon_path used to collapse to "." and
  report "tool not found"; it now stays blank so auto-detection (and
  the bundled FFmpeg) is used.

Tests
- Scan-disc process test fakes gained poll() + stdout.close().
- Full suite: 1647 passed, 5 skipped.

### What's NOT in this release

No new ripping/validation/organization features.  These are
targeted fixes to scan responsiveness and tool detection.
