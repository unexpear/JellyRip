### What's New in 1.0.9

Safety
- ask_yesno now has a 300s safety timeout and abort check (was unbounded).
- ask_input serialised with a lock — concurrent prompts no longer clobber each other.

Reliability
- Version printed at start of every rip session in log for unambiguous debug.
- rip_selected_titles returns failure on per-title failures, not just full abort.
- Session state machine reset and completed correctly in TV/movie disc flow.

Security
- Auto-update uses unique temp dir per download (prevents TOCTOU).
- PowerShell signature check uses -LiteralPath + param block (no path injection).

Code quality (1.0.8 + 1.0.9)
- _normalize_rip_result scans MKV files recursively (catches MakeMKV subdirs).
- get_available_drives has 30s timeout (stalled MakeMKV no longer hangs app).
- check_disk_space no longer creates directories as side effect.
- Corrupt config now logs visible warning instead of silently resetting.
- Wildcard imports replaced with explicit imports throughout.
- handle_fallback simplified, hasattr duck-checks removed.
- DummyGUI in tests has all required stub methods.
