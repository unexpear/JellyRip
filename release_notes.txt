## JellyRip v1.0.21 Release Notes (UNSTABLE)

JellyRip v1.0.21 — audit-driven cleanup.  Adds a configurable
drive-probe retry count, picks the 64-bit MakeMKV binary on x64
Windows, and trims a stale empty file from the bundle.  Engine
behavior under default settings is unchanged from v1.0.20.

### What's New in 1.0.21

Engine
- Drive-probe retries are now config-driven via
  opt_drive_probe_retries (default 3) and
  opt_drive_probe_backoff_seconds (default 2.0).  Users with slow
  trays can bump these without code changes.  Default behavior
  matches the prior hardcoded 3-retries-with-2-4-8 backoff exactly.
- 64-bit MakeMKV binary preferred on x64 Windows.  _DEFAULT_MAKEMKVCON
  picks makemkvcon64.exe when ProgramW6432 is set, falls back to
  makemkvcon.exe on 32-bit / non-Windows.  Avoids the rare
  mixed-bitness "process suspended" hang.

Bundle / repo
- Removed gui_qt/qss/warm.qss (empty 0-byte placeholder).  Was
  filtered at load time but still shipping in the bundle.
- pyproject.toml keywords dropped tkinter, added pyside6 + qt.

Documentation
- README "shipped path as of v1.0.0" -> "since v1.0.19" (the Qt-only
  milestone).

### What's NOT in this release

No UI or workflow changes.  Engine behavior under default settings
is byte-identical to v1.0.20.
